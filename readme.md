# TheCollapse

JavaScript / HTML / CSS · Node.js · Docker

---

## 概要

**TheCollapse** は、チャット・履歴管理・アカウント機能・ミニゲームを備えた Web アプリケーションです。  
Node.js をバックエンドとし、Docker による環境構築に対応しています。

---

## 技術スタック

| 分類 | 内容 |
|------|------|
| フロントエンド | HTML / CSS / JavaScript |
| バックエンド | Node.js (index.js) |
| インフラ | Docker |

---

## セットアップ

### Docker を使う場合

```bash
docker build -t thecollapse .
docker run -p 3000:3000 thecollapse
```

### ローカルで直接起動する場合

```bash
npm install
node index.js
```

## 更新履歴

### v1.3.5

#### 🏷️ バージョン表示の自動化
- トップページのバージョンバッジを `readme.md` の「更新履歴」内で**一番上に記載されたバージョン**から自動取得するようにした。今後はこのファイルを更新するだけでサイト上の表示が追従する。

#### 💬 チャットの修正・改善
- **送信できない不具合の修正**: アイコン付きアカウントで `[ws error] Max payload size exceeded` が発生し、メッセージが送信できなくなる問題を修正。WebSocket の最大フレームサイズをアイコン（data URL）込みでも扱えるサイズへ引き上げ、テキスト本文の長さ制限とは独立して管理するようにした。
- **デザイン調整**: メッセージのアバター/余白・入力欄・返信バー・リアクションまわりの細部を整え、視認性と一体感を向上。

#### 🔐 ログインの利便性向上
- **ユーザー名の大文字小文字を区別しない**: ログイン時にユーザー名の大文字小文字どちらでも認証できるようにした（登録名はそのまま保持・表示）。重複登録も大文字小文字を無視して防止する。

### v1.3.4

#### 💬 チャット機能の拡張
- **メッセージ編集機能**: 自分のメッセージを編集できるようにした。編集可否はサーバー側で送信者本人かを確認（admin であっても他人の発言は改変不可）。編集後は「(編集済み)」バッジを表示。
- **メッセージへの返信（リプライ）**: 引用返信に対応し、会話の流れを明確化。引用元の本文・送信者名はサーバーが元メッセージから確定し、なりすましや細工本文の混入を防止。引用部クリックで返信元へジャンプ・ハイライト。
- **絵文字リアクション**: メッセージにリアクションを付けられるようにした（CSS のみで未実装だった機能を実装）。ホワイトリスト方式の絵文字のみ許可し、同一ユーザーの再押下でトグル解除。1 メッセージあたりのリアクション種類数に上限を設けて肥大化・DoS を抑止。
- **メッセージ削除確認**: 削除前に確認ダイアログを表示し、誤削除を防止。

### v1.3.1

#### 🔐 アカウント機能の脆弱性修正（重大なセキュリティ修正リリース）
##### 修正した脆弱性
- **パスワードハッシュの全公開**: `worksheets/data/<username>.json` および `account.json` が静的配信で誰でも GET でき、`passwordHash` が漏洩していた → データファイルは**ログイン中の本人のみ**アクセス可能にし、認証情報は静的配信外の `auth-data/auth.json` でサーバーが一元管理。
- **弱いハッシュ方式（djb2 / 32bit・ソルト無し）**: 容易に逆算可能だった → **scrypt + ランダムソルト**へ変更。
- **クライアント側のみの認証**: パスワード照合がブラウザ内のみで、誰でも任意ユーザーのデータファイルを `PUT` で上書き（なりすまし・改ざん）できた → サーバー側で**セッション（HttpOnly Cookie）による認証・認可**を実装。書き込みは本人のファイルのみ許可。
- **機微フィールドのデータ混入**: データファイルへ `passwordHash` / `admin` 等を保存できた → サーバー側でこれらのフィールドを保存時に除去。
- **アカウント列挙**: ログイン応答の差異でユーザー有無が判別できた → 応答時間とメッセージを均一化。

### v1.3

#### 🛡️ 脆弱性修正（セキュリティ強化リリース）
既知の脆弱性を網羅的に洗い出し、可能な限り対処した安全性重視のリリース。

##### ⚠️ 既知の注意点
- `wisp-server-node` は upstream で deprecated 警告が出ているが、後継 `@mercuryworkshop/wisp-js` への移行は API 差分が大きく、プロキシ機能の互換性確認が別途必要なため本リリースでは見送り。
- プロキシ用途のアプリケーションのため、SSRF 的なアクセス自体は機能要件であり制限していません。設置者側でネットワーク的なアクセス制御を行うことを推奨。
